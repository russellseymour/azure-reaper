Function Upgrade

New releases of Azure Reaper will be made available periodically which will provide bug fixes and new features. It is recommended that the releases are monitored to determine if an upgrade is necessary.

The commands shown here will upgrade the function to version 0.6.18. In order to perform the upgrade the following information is required. (The values in brackets are the ones that are used in the example commands).

  • Resource Group (azreaper-test-15) - name of the resource group that Azure Reaper has been deployed into.
  • Function Name (azreaper-wlw5) - name of the Azure function where the code is deployed to.

The commands shown here only upgrade the code running in the Azure Function and do not touch any of the architecture. This means that Azure Reaper can be upgrade in situ without loosing existing settings or subscriptions.

# Download the function zip file from the release page
curl https://github.com/russellseymour/azure-reaper/releases/download/0.6.18/azure-reaper-0.6.18.zip -o azure-reaper-0.6.18.zip

# Ensure logged into Azure
az login

# Deploy the updated function
az functionapp deployment source config-zip -g azreaper-test-15 -n azreaper-wlw5 --src azure-reaper-0.6.18.zip

# Download the function zip file from the release page
Invoke-RestMethod -Uri https://github.com/russellseymour/azure-reaper/releases/download/v0.6.18/azure-reaper-0.6.18.zip -Outfile azure-reaper-0.6.18.zip

# Ensure logged into Azure
Connect-AZAccount

# Deploy the updated function
Publish-AzWebApp -ResourceGroupName azreaper-test-15 -Name azreaper-wlw5 -ArchivePath azure-reaper-0.6.18.zip
